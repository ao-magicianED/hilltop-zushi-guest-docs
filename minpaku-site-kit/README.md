# 🏡 民泊ハウスガイド制作キット（House Guide Kit）

**プログラミング不要**で、ゲストが滞在中に読む「**多言語ハウスガイド（電子マニュアル）**」が作れるキットです。
チェックイン方法・Wi-Fi・ゴミ出し・設備の使い方・ハウスルール・緊急時の対応…を、**スマホできれいに読める1ページ**にまとめられます。

> このキットは、実際に運営している民泊「Hilltop Zushi」のハウスガイドで使ったノウハウを、
> だれでも再利用できる形にまとめたものです。
> **公開用に、物件情報はすべて架空の見本（Aomine Lodge）に差し替えています。**

---

## ✨ このハウスガイドの特徴

| 特徴 | かんたんに言うと |
|---|---|
| 📄 **ファイル1つで完成** | `.html` を1個ダブルクリックすれば開く。サーバーも環境構築も不要 |
| 🌍 **5言語に対応** | 日本語・英語・中国語・韓国語・フランス語をボタン1つで切替（外国人ゲスト対応） |
| 📑 **サイドバー目次** | 「Wi-Fi」「ゴミ」「緊急時」など、知りたい項目へワンタップで移動 |
| 🚦 **色分けブロック** | 緊急=赤／注意=橙／情報=青／OK=緑 で、大事なことが直感的に伝わる |
| 🔢 **暗証番号・Wi-Fi枠** | 鍵・ネットなど「ゲストが最初に困ること」をすぐ解決 |
| 📱 **スマホ最適＆QR配布** | QRコードにして冷蔵庫・玄関に貼れる。予約メッセージにURLを載せるだけ |
| 🖼 **写真プレースホルダ** | 写真が未用意でも「ここに貼る」場所が分かる枠を表示 |

※ **フレームワーク／ビルド** … プロが使う組み立てツールのこと。初心者には難しいので、このキットは**あえて使わず**、`.html`ファイル単体で動くようにしています。

## 📦 中身（ファイルの説明）

```
minpaku-site-kit/
├─ index.html             ← 最初に開く入口ページ（見本・テンプレへの入口）
├─ README.md              ← いま読んでいるこの説明書
├─ PROMPTS.md             ← AIにハウスガイドを作らせる「コピペ用の指示文」集 ★おすすめ
├─ template-manual.html   ← 白紙のテンプレート（穴埋めで作る用）
├─ example-manual.html    ← 完成見本（架空の湖畔ヴィラ「Aomine Lodge」のハウスガイド）
└─ _archive-lp/           ← おまけ：集客用のLP（ランディングページ）版テンプレ
```

> 💡 **「ハウスガイド」と「集客LP」の違い**
> - **ハウスガイド**（このキットの主役）＝ 予約した後・**滞在中**にゲストが読む「説明書」
> - **集客LP**（`_archive-lp/`）＝ 予約**前**に見せる「お店の顔」。必要な人はこちらもどうぞ

## 🚀 まずは見てみる（30秒）

1. `example-manual.html` を**ダブルクリック**する
2. ブラウザ（Chrome等）で完成見本のハウスガイドが開きます
3. 右上の **言語ボタン（日 / EN / 中 / 한 / FR）** で切り替わるのを試す
4. 左の**サイドバー目次**で「Wi-Fi」「緊急時」などへジャンプしてみる

> 見本の写真は「仮の枠」、物件情報は「架空のサンプル」です。実際は自分の情報に差し替えます。

---

## 🛠 自分の物件用に作る（3ステップ）

まず `template-manual.html` を**コピー**し、ファイル名を `house-guide.html` などに変えてから編集します（原本は残しておくと安心）。

### ステップ1️⃣：情報を入れる
- コピーした `house-guide.html` を「メモ帳」や「VS Code」で開く
- 各セクションの **`【　】` で囲まれた部分**（物件名・住所・Wi-Fi・ルールなど）を自分の物件の情報に書き換える

### ステップ2️⃣：QRコードにする
- 後述の方法でネットに公開し、出てきたURLを**QRコード（無料サイト）**に変換する

### ステップ3️⃣：貼る・送る
- QRを**冷蔵庫・玄関・各部屋**に貼る／**予約サイトのメッセージ**にURLを載せる
- ゲストはスマホで読み込むだけで、自分の言語でガイドを読めます

> 💡 **むずかしければ、ステップ1を全部AIにやってもらえます。** → 次へ

## ✅ 公開前チェックリスト（だいじ・とくに機密）

ハウスガイドには**Wi-Fiパスワードや鍵の暗証番号**など、扱いに注意が要る情報が入ります。公開前に必ず確認しましょう。

- [ ] **Wi-Fiパスワード・暗証番号**を載せるか決めたか（→ 下の「機密の扱い」を参照）
- [ ] 物件名・住所・最寄り駅は自分のものになっているか
- [ ] **緊急連絡先・最寄りの病院・避難場所**が**実在の正しい情報**になっているか
- [ ] 5言語すべてで差し替え漏れがないか（中国語・韓国語・仏語も）
- [ ] サンプルの文章（「※サンプル」「Aomine」など）が残っていないか
- [ ] スマホで開いて、文字切れ・崩れがないか
- [ ] 写真が重すぎないか（横1600px程度に縮小）

---

## 🔒 機密の扱い（ハウスガイドならではの注意）

ハウスガイドは「**滞在中のゲストだけ**が見る」のが前提です。Wi-Fiや暗証番号を載せるなら、URLの配り方に気をつけてください。

- **おすすめ**：URLを**むやみに公開せず**、QRコード＋予約サイトのメッセージで**ゲストにだけ**共有する
- 検索に出したくない場合は、推測されにくいファイル名／URLにする
- 暗証番号などは「予約メッセージで別途お伝えします」とガイド側に書き、**番号自体はガイドに載せない**運用も安全
- **緊急・防災情報**（病院・避難所）は、見本は架空です。**必ず実在の正確な情報に**差し替えてください

## 🌐 インターネットに無料で公開する＋QR化

作った `.html` を、ゲストがスマホで開けるようにする方法です。**いちばん簡単な順**に紹介します。

### 方法A：ドラッグ&ドロップで公開（初心者おすすめ・GitHub不要）
1. [Netlify Drop](https://app.netlify.com/drop) または [Cloudflare Pages](https://pages.cloudflare.com/) を開く
2. サイトのフォルダを**画面にドラッグ&ドロップ**する
3. 数十秒で `https://〇〇〇.netlify.app` のような**公開URL**が発行される

### 方法B：GitHub Pages（このキットをGitHubで受け取った人向け）
1. GitHubで自分のアカウントにこのリポジトリを置く
2. リポジトリの **Settings → Pages** を開く
3. 公開元のブランチを選んで保存 → 数分で公開URLが出ます

### QRコードにする
- 「QRコード 作成 無料」で検索して出るサイトに、公開URLを貼るだけ
- 出てきたQRを印刷して、**冷蔵庫・玄関・各部屋**に貼る

## 📸 写真のコツ

- **横向き（横長）** の写真を使う（縦写真は切れて見えます）
- 各セクション（設備・BBQ・ゴミ置き場など）は、**実際の場所が分かる写真**だとゲストが迷いません
- スマホ撮影でも十分。明るさを少し上げるだけで見違えます
- 容量が大きすぎると重いので、**横1600px程度**に縮小すると◎

## ❓ よくある質問

**Q. プログラミングの知識は必要？**
A. 不要です。`【　】`の中を書き換えるだけ。むずかしければAIに頼めます。

**Q. 予約サイト（Airbnb等）と何が違う？**
A. これは**滞在中の説明書**です。予約前に見せる「集客LP」は `_archive-lp/` に同梱しています。

**Q. 紙にも貼りたい**
A. 公開URLをQRコードにして掲示するのがおすすめです。A4印刷用の掲示版が必要なら別途作れます。

**Q. 何語まで対応できる？**
A. 見本は5言語（日英中韓仏）。AIで他の言語も追加できます（→ PROMPTS.md ④）。

**Q. お金はかかる？**
A. キットも公開も無料です。独自ドメインが欲しい場合だけ年1,000〜2,000円ほど。
